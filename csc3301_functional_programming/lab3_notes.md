# Lab 3 Nottes: Higher-order Functions
- Defination: A higher-order function either takes functions as arguments or returns a functionas its result.


Examples: map, filter, reduce, sorted(key=func).

Patterns:
    **apply_twice(f, x): ** demonstartes passing functions as arguemnts.
    **make_adder(n):** demonstartes returning returning functions (factories).
    **repeat(n, action): ** shows callbacks and separation of concerns.
